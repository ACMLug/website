1. Introduction

NOTE: Many examples and explanations in this article were taken from Advanced Programming in the Unix Environment by W. Richard Stevens, (c) 1992.

1.1 Named Pipes (FIFOs)

FIFOs, sometimes called named pipes, are an UNIX interprocess communication mechanism that is a standard part of POSIX.1. While standard pipes can only be used between related processes when a common ancestor has created the pipe, FIFOs allow unrelated processes to exchange data. A FIFO is actually a type of file, and the standard file I/O functions (open, close, read, write, unlink, etc.) all work with FIFOs.

As you will see, using a FIFO for your .signature file instantly allows every application access to a randomly created .signature, without requiring modifications to the individual applications. To them, your .signature file looks and acts just like a regular text file.

1.2 Basic Strategy for Writing a Signature Randomizer

Since FIFOs look and act just like regular files, we can create a randomized .signature by simply making .signature into a FIFO and having a process deliver data to it upon request. To do this, we can write a daemon that will live "forever" and constantly tries to send a new, random signature to the FIFO.
